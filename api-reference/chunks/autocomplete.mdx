## Authentication

Requires API key with **readonly** role or higher for the specified dataset's organization.

## How Autocomplete Works

1. **Prefix Matching**: First attempts to find chunks where words start with the query text
2. **Extended Search**: If `extend_results` is true and no prefix matches are found, falls back to semantic or full-text search
3. **Highlighting**: Results are highlighted to show where matches occur
4. **Performance**: Optimized for low-latency responses suitable for real-time search-as-you-type

## Use Cases

- **Search bars**: Build responsive search-as-you-type experiences
- **Command palettes**: Quick navigation and command completion
- **Tag suggestions**: Suggest tags as users type
- **Entity completion**: Auto-complete entity names, products, or categories

## Best Practices

- Use `page_size` of 5-10 for typical autocomplete dropdowns
- Set `slim_chunks: true` for faster response times
- Enable `extend_results` to provide fallback results
- Use `highlight_options` to emphasize matched portions
- Consider adding debouncing on the client side to reduce API calls
