## Overview

This quickstart will guide you through setting up Trieve and making your first search request. You'll create a dataset, upload some content, and perform a search query.

<Info>
  **Time to complete**: ~5 minutes
</Info>

## Prerequisites

- A Trieve account ([sign up free](https://dashboard.trieve.ai))
- Node.js 16+ installed (for TypeScript SDK examples)
- Basic familiarity with REST APIs or TypeScript

## Step 1: Create an Organization and Dataset

<Steps>
  <Step title="Sign up for Trieve">
    Navigate to [dashboard.trieve.ai](https://dashboard.trieve.ai) and create your account. You'll start with 1,000 free chunks.
  </Step>

  <Step title="Create your first dataset">
    After signing in, create a new dataset from the dashboard. This will give you:
    - A **Dataset ID** (UUID format)
    - An **Organization ID** (UUID format)
  </Step>

  <Step title="Generate an API key">
    Navigate to your organization settings and create an API key:
    - Give it a descriptive name (e.g., "Development Key")
    - Set the role to **Admin** for full access
    - Copy and save the API key securely (you won't see it again)
  </Step>
</Steps>

<Warning>
  Store your API key securely! It provides full access to your Trieve organization.
</Warning>

## Step 3: Initialize the Client

<CodeGroup>
```typescript TypeScript
import { TrieveSDK } from "trieve-ts-sdk";

const trieve = new TrieveSDK({
  apiKey: "tr-********************************",
  datasetId: "your-dataset-id-here",
});
```

```python Python
import trieve_py_client
from trieve_py_client.rest import ApiException

configuration = trieve_py_client.Configuration(
    host = "https://api.trieve.ai"
)

# Configure API key authorization
configuration.api_key['ApiKey'] = 'tr-********************************'

api_client = trieve_py_client.ApiClient(configuration)
```

```bash cURL
# Set your credentials as environment variables
export TRIEVE_API_KEY="tr-********************************"
export TRIEVE_DATASET_ID="your-dataset-id-here"
```
</CodeGroup>

## Step 4: Upload Your First Chunk

A "chunk" is a piece of searchable content in Trieve. Let's upload some sample data:

<CodeGroup>
```typescript TypeScript
const chunk = await trieve.createChunk({
  chunk_html: "<p>Trieve is an all-in-one infrastructure for search, recommendations, and RAG. It combines semantic search, full-text search, and recommendations into a single API.</p>",
  link: "https://trieve.ai",
  metadata: {
    title: "What is Trieve?",
    category: "introduction",
  },
  tracking_id: "trieve-intro-001",
});

console.log("Chunk created:", chunk);
```

```python Python
chunk_api = trieve_py_client.ChunkApi(api_client)

chunk_data = trieve_py_client.CreateChunkReqPayloadEnum(
    chunk_html="<p>Trieve is an all-in-one infrastructure for search, recommendations, and RAG. It combines semantic search, full-text search, and recommendations into a single API.</p>",
    link="https://trieve.ai",
    metadata={
        "title": "What is Trieve?",
        "category": "introduction",
    },
    tracking_id="trieve-intro-001"
)

try:
    chunk = chunk_api.create_chunk(
        chunk_data,
        tr_dataset="your-dataset-id-here"
    )
    print("Chunk created:", chunk)
except ApiException as e:
    print("Exception:", e)
```

```bash cURL
curl -X POST "https://api.trieve.ai/api/chunk" \
  -H "Authorization: Bearer $TRIEVE_API_KEY" \
  -H "TR-Dataset: $TRIEVE_DATASET_ID" \
  -H "Content-Type: application/json" \
  -d '{
    "chunk_html": "<p>Trieve is an all-in-one infrastructure for search, recommendations, and RAG. It combines semantic search, full-text search, and recommendations into a single API.</p>",
    "link": "https://trieve.ai",
    "metadata": {
      "title": "What is Trieve?",
      "category": "introduction"
    },
    "tracking_id": "trieve-intro-001"
  }'
```
</CodeGroup>

<Tip>
  **Tracking IDs** are optional but recommended. They let you reference chunks using your own identifiers from external systems.
</Tip>

## Step 6: Perform Your First Search

Now let's search for content using different search types:

<CodeGroup>
```typescript TypeScript
// Hybrid search (recommended for best results)
const results = await trieve.search({
  query: "How does Trieve search work?",
  search_type: "hybrid",
  page_size: 10,
  highlight_options: {
    highlight_delimiters: ["<mark>", "</mark>"],
  },
});

console.log("Search results:", results);

// Print formatted results
results.chunks.forEach((chunk, index) => {
  console.log(`\n${index + 1}. ${chunk.metadata?.title || 'Untitled'}`);
  console.log(`   Score: ${chunk.score}`);
  console.log(`   Content: ${chunk.chunk_html}`);
});
```

```python Python
search_api = trieve_py_client.ChunkApi(api_client)

search_data = trieve_py_client.SearchChunksReqPayload(
    query="How does Trieve search work?",
    search_type="hybrid",
    page_size=10,
    highlight_options={
        "highlight_delimiters": ["<mark>", "</mark>"]
    }
)

try:
    results = search_api.search_chunks(
        search_data,
        tr_dataset="your-dataset-id-here",
        x_api_version="V2"
    )
    
    for i, chunk in enumerate(results.chunks):
        print(f"\n{i + 1}. {chunk.metadata.get('title', 'Untitled')}")
        print(f"   Score: {chunk.score}")
        print(f"   Content: {chunk.chunk_html}")
except ApiException as e:
    print(f"Exception: {e}")
```

```bash cURL
curl -X POST "https://api.trieve.ai/api/chunk/search" \
  -H "Authorization: Bearer $TRIEVE_API_KEY" \
  -H "TR-Dataset: $TRIEVE_DATASET_ID" \
  -H "X-API-VERSION: V2" \
  -H "Content-Type: application/json" \
  -d '{
    "query": "How does Trieve search work?",
    "search_type": "hybrid",
    "page_size": 10,
    "highlight_options": {
      "highlight_delimiters": ["<mark>", "</mark>"]
    }
  }'
```
</CodeGroup>

### Try Different Search Types

<Tabs>
  <Tab title="Hybrid">
    ```typescript
    // Best of both worlds: semantic + full-text
    const results = await trieve.search({
      query: "vector embeddings",
      search_type: "hybrid",
    });
    ```
    Combines semantic and full-text search with re-ranking for optimal results.
  </Tab>
  <Tab title="Semantic">
    ```typescript
    // Conceptual similarity using embeddings
    const results = await trieve.search({
      query: "finding similar content",
      search_type: "semantic",
    });
    ```
    Uses dense vector embeddings to find conceptually related content.
  </Tab>
  <Tab title="Full-Text">
    ```typescript
    // Keyword-based with typo tolerance
    const results = await trieve.search({
      query: "SPLADE neural search",
      search_type: "fulltext",
    });
    ```
    Neural sparse-vector search with typo tolerance and synonym understanding.
  </Tab>

## Common Issues

<AccordionGroup>
  <Accordion title="No search results returned" icon="circle-question">
    - **Wait for indexing**: Chunks need a few seconds to be indexed after upload
    - **Check your dataset ID**: Ensure you're searching the correct dataset
    - **Verify chunk upload**: Use the dashboard to confirm chunks were created
    - **Try different search types**: Semantic vs full-text may yield different results
  </Accordion>

  <Accordion title="Authentication errors" icon="lock">
    - **Check your API key**: Ensure it starts with `tr-` and is complete
    - **Verify headers**: Both `Authorization` and `TR-Dataset` headers are required
    - **Check permissions**: Your API key needs appropriate role permissions
    - See the [Authentication guide](/authentication) for details
  </Accordion>

  <Accordion title="Rate limiting" icon="gauge">
    - Free tier includes 1,000 chunks and reasonable rate limits
    - Contact support for higher limits: developers@trieve.ai
    - Consider batch operations to reduce API calls
  </Accordion>
</AccordionGroup>
