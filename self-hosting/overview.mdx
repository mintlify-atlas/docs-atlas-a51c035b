## Introduction

Trieve can be fully self-hosted in your VPC or on-premises infrastructure, giving you complete control over your data and deployment. This guide covers the different deployment options available.

## Architecture Components

A complete Trieve deployment consists of the following services:

### Core Services

- **Server** - Main API server handling requests
- **Workers** - Background job processors for ingestion, deletion, and file processing
- **Dashboard** - Administrative web interface
- **Search UI** - Search interface
- **Chat UI** - Conversational interface

### Data Stores

- **PostgreSQL** - Primary relational database for metadata
- **Qdrant** - Vector database for embeddings and similarity search
- **Redis** - Cache and job queue
- **ClickHouse** - Analytics database
- **S3/MinIO** - Object storage for files

### Supporting Services

- **Keycloak** - Identity and access management (OIDC provider)
- **Tika** - Document parsing and text extraction
- **Embedding Services** - Text embedding models (Jina, SPLADE, BGE-M3, Reranker)

## System Requirements

### Minimum Requirements (Development)

- **CPU**: 4 cores
- **RAM**: 16 GB
- **Storage**: 50 GB
- **OS**: Linux, macOS, or Windows with WSL2

### Recommended Requirements (Production)

- **CPU**: 8+ cores
- **RAM**: 32+ GB
- **Storage**: 200+ GB SSD
- **GPU**: Optional - NVIDIA T4 or L4 for faster embedding generation
- **Network**: High bandwidth for API traffic

## Cloud Provider Support

Trieve includes Terraform configurations for automated deployment on:

- **Google Cloud Platform (GCP)** - GKE with GPU node pools
- **Amazon Web Services (AWS)** - EKS deployment
- **Microsoft Azure** - AKS deployment
